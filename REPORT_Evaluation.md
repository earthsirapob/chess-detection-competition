## 1. วิธีการวัดผล (Evaluation Metrics)
การทดสอบประสิทธิภาพของระบบวัดจากการเปรียบเทียบ PGN Move ที่ระบบสร้างขึ้น (Generated PGN) กับ Ground Truth ที่ถูกต้อง โดยพิจารณาตัวชี้วัดดังนี้:

- **Correct Moves**: จำนวนตาเดินที่ระบุได้ถูกต้องทั้งตัวหมาก ตำแหน่งเริ่มต้น และตำแหน่งปลายทาง
- **Move Accuracy (%)**: (จำนวน Moves ที่ถูก / จำนวน Moves ทั้งหมด) * 100
- **Game Completion**: ความสามารถในการบันทึกเกมจนจบโดยไม่มีข้อผิดพลาดที่ทำให้เกมหยุดชะงัก (Illegal Move Error)

## 3. การวิเคราะห์ความผิดพลาด (Error Analysis)
จากการสังเกตผลลัพธ์ พบปัจจัยหลักที่ส่งผลต่อความแม่นยำ:

### 3.1 ปัญหาแสงเงา (Lighting & Shadows)
- ในวิดีโอบางช่วงที่มีเงาพาดผ่านกระดาน (เช่น แขนผู้เล่นบังไฟ) อาจทำให้ Model ตรวจจับสีของหมากผิดพลาด (ขาวเป็นดำ) ได้บ้างในบางเฟรม
- **การแก้ไข**: ใช้ Voting Mechanism (Majority Vote 15 เฟรม) เข้ามาช่วยกรองค่าที่ผิดพลาดชั่วคราวออกไปได้มาก

### 3.2 การบดบัง (Occlusion)
- จังหวะที่มือของผู้เล่นหยิบหมากและบังหมากตัวอื่น ทำให้ระบบมองไม่เห็นหมากชั่วคราว
- **การแก้ไข**: ระบบจะรอให้สถานะ "นิ่ง" (Stable) เกิน 30 เฟรม (ประมาณ 1 วินาที) หลังจากมือออกไปแล้ว ค่อยประมวลผลการเดิน ทำให้ไม่เกิด False Alarm ตอนมือบัง

## 4. สรุปผล (Conclusion)
ระบบสามารถตรวจจับและบันทึกเกมหมากรุกจากวิดีโอได้ในระดับที่น่าพอใจ โดยเฉพาะในสภาพแวดล้อมที่มีการควบคุมแสงและมุมกล้อง การใช้เทคนิค **Grid Projection** ช่วยให้รองรับมุมกล้องที่เอียงได้ดีกว่าการใช้ตารางแบบ Static grid และ Logic การตรวจจับ **Rules-based Validation** ช่วยกรองท่าเดินที่ผิดกติกาออกไปได้อย่างมีประสิทธิภาพ
